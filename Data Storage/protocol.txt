==== Data Storage protocol v1.0 ====
Request
{
  "session": "id",
  "bank": "source_raw",
  "operation": "post",
  "data": {}
}
session - идентификатор сессии, который выдаётся менеджером
bank - имя раздела данных. Например, "source_raw" - исходный код программы до обработки лексером. 
operation - post или get - добавить или получить данные. Удалять данные нельзя, можно только добавлять и читать.
data - сами данные. Это поле нужно только для операции добавления. 

Answer
{
  "success": true,
  "data": {}
}
success - true или false - успешность выполнения запроса
data - данные, если была запрошена операция получения данных


!!! Есть два МАГИЧЕСКИХ запроса (могут помочь при отладке). Они вызываются менеджером и не при каких условиях не должен быть использован в сервисах.

Этот запрос создаёт новую сессию или очищает имеющуюся по id.
{
  "session": "id",
  "bank": "any",
  "operation": "post",
  "data": "Manager's Mandat: Create Session"
}

Этот запрос удаляет имеющуюся сессию по id.
{
  "session": "id",
  "bank": "any",
  "operation": "post",
  "data": "Manager's Mandat: Remove Session"
}
